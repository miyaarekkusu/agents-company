# 初期設定メモ（開発環境セットアップ手順）

このプロジェクト（AIエージェントお仕事ゲーム）を新しい開発環境でセットアップする人向けの手順です。

## 0. 前提ツール

以下がインストール済みであることを確認してください。

- Node.js 24系（`frontend/package.json` の devDependencies が `@types/node ^24.x` 前提）
- Python 3.13以降（`backend-core/pyproject.toml` の `requires-python = ">=3.13"`）
- Docker Desktop（Docker Compose込み。まとめて起動する場合に使用）
- git

## 1. まずAGENTS.mdを読む

リポジトリをcloneしたら、作業を始める前に必ずリポジトリ直下の `AGENTS.md` を読んでください。
プロジェクトの企画意図・ゲームシステム・技術スタック・ディレクトリ構成方針がまとまっています。
実装・作業内容がAGENTS.mdの方針と矛盾しないように進めることが前提になっています。

```
git clone <このリポジトリのURL>
cd ai-company
# ここでAGENTS.mdを読む
```

## 2. frontend側のセットアップ

React（Vite + TypeScript）構成です。

```
cd frontend
npm install
copy .env.example .env   # PowerShellの場合。bash系なら cp .env.example .env
```

- `.env` の中身（`VITE_API_BASE_URL` など）は必要に応じて環境に合わせて書き換えてください。
- 開発サーバー起動: `npm run dev`（Vite、デフォルトで http://localhost:5173 ）
- Lint: `npm run lint`（oxlint）
- ビルド: `npm run build`

## 3. backend-core側のセットアップ

FastAPI（Python）構成です。

```
cd backend-core
python -m venv .venv
```

venvの有効化（OSによって異なります）:

```
# Windows PowerShell
.venv\Scripts\Activate.ps1

# Windows cmd
.venv\Scripts\activate.bat

# macOS / Linux
source .venv/bin/activate
```

有効化した状態で依存関係をインストールし、`.env` を用意します。

```
pip install -e .
copy .env.example .env   # PowerShellの場合。bash系なら cp .env.example .env
```

- `.env` にはDB接続情報（`DATABASE_URL` など）を設定します。ローカルでPostgreSQLを別途用意しない場合は、後述のDocker Composeで `db` サービスだけ起動する方法もあります。
- 開発サーバー起動確認は `memo/バックエンド確認方法.md` を参照してください。

## 4. Docker Composeでまとめて起動する（選択肢）

frontend / backend-core / db（PostgreSQL）をまとめて立ち上げたい場合は、リポジトリ直下の `docker-compose.yml` を使ってDocker Composeでまとめて起動することもできます。

```
docker compose up -d
```

- `db`: PostgreSQL（postgres:18）
- `backend-core`: FastAPI（ポート8000で公開）
- `frontend`: Vite（ポート5173で公開）

ローカルでnpm/venvを個別にセットアップせず、まずは動作確認だけしたい場合はこちらが手軽です。
停止する場合は `docker compose down` を実行してください。
