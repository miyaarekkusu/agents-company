# バックエンド確認方法メモ（backend-core / FastAPI）

backend-core（FastAPI）が正しく動作しているかを確認する手順です。

## 方法1: ローカル（venv）で直接起動して確認する

`memo/初期設定.md` の手順でvenvを作成・有効化し、`pip install -e .` まで完了している前提です。

```
cd backend-core
uvicorn app.main:app --reload --port 8000
```

起動したら、別のターミナルで以下を実行します。

```
curl http://localhost:8000/health
```

以下のレスポンスが返ってくれば正常に動作しています。

```
{"status":"ok"}
```

## 方法2: Docker Compose経由で確認する

```
docker compose up -d db backend-core
```

起動したら、同じく以下でヘルスチェックします。

```
curl http://localhost:8000/health
```

`{"status":"ok"}` が返ればOKです。

ログを確認したい場合は以下を実行します。

```
docker compose logs backend-core
```

（リアルタイムで追いたい場合は `docker compose logs -f backend-core`）

## 方法3: `POST /api/tasks` を実際に叩いて確認する（curl経由）

LangGraph経由でDeepSeek APIが正しく呼べているかを確認します。方法1か方法2でサーバーを起動した状態で行います。

Windows/Git Bashで日本語をそのまま`curl -d`に書くと文字化けすることがあるため、一度ファイルに書き出してから送ります。

```
printf '%s' '{"task": "こんにちは、自己紹介を一文でしてください。"}' > req.json
curl -X POST http://localhost:8000/api/tasks -H "Content-Type: application/json" --data-binary @req.json
```

`{"result": "..."}` に日本語の応答が入っていればOKです。

## 方法4: ターミナルで直接対話して確認する（`scripts/chat.py`）

サーバー起動・curlなしで、LangGraphのグラフを直接呼び出してターミナル上で対話しながら確認できる簡易CLIです。プロンプトや会話内容を素早く試したいときはこちらが手軽です。

```
cd backend-core
.venv\Scripts\Activate.ps1   # venv有効化（PowerShellの場合）
python scripts/chat.py
```

`お題>` に日本語で質問を入力してEnterを押すと、DeepSeekからの応答が表示されます。何も入力せずEnterで終了します。
（`app/core/config.py` は `.env` を絶対パスで読み込むようにしているため、`backend-core`以外のディレクトリから実行しても動作します。）

## うまく動かない場合のチェックポイント

- **.envの設定漏れ**: `backend-core/.env` が `.env.example` からコピーされているか、`DATABASE_URL` 等の必須項目が設定されているか確認する。
- **ポート競合**: 8000番ポート（backend-core）や5432番ポート（db）を他のプロセスが既に使っていないか確認する。
- **DBが起動しているか**: `docker compose ps` で `db` サービスが `healthy` になっているか確認する。ローカル（venv）起動の場合は、別途PostgreSQLが起動している必要がある（Docker Composeで `db` サービスだけ先に起動しておくのも手）。
- **依存関係の入れ忘れ**: ローカル起動の場合、venvが有効化された状態で `pip install -e .` が完了しているか確認する。
- **CORSエラー（フロントから叩く場合）**: `CORS_ORIGINS` の設定がフロントのオリジン（例: `http://localhost:5173`）と一致しているか確認する。
- **DeepSeekが自分をClaude等と名乗ることがある**: `/api/tasks` や `scripts/chat.py` の応答で、モデルが「私はAnthropicのClaudeです」のように別のAIを名乗ることがある。これは通信・実装のバグではなく、モデル側の自己認識が学習データの影響で混乱する既知の現象。エージェントに正しい自己認識を持たせたい場合は、systemプロンプトで明示的に「あなたはDeepSeekベースの◯◯エージェントです」等と伝える必要がある。
