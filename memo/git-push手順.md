# git push手順メモ（このプロジェクトでの統一手順）

リモートリポジトリに変更をpushするときは、必ず以下の順序・コマンドのとおりに実行してください。

## 手順

1. ```
   git add .
   ```
   変更したファイルをすべてステージング（コミット対象に追加）する。

2. ```
   git commit -m "<コミットメッセージ>"
   ```
   ステージした変更をコミットする。メッセージには変更内容が分かる説明を書く。

3. ```
   git checkout -b <ブランチ名>
   ```
   新しいブランチを作成して切り替える。ブランチ名は変更内容が分かる名前にする（例: `feature/xxx`, `docs/xxx` など）。

4. ```
   git push origin <ブランチ名>
   ```
   作成したブランチをリモートリポジトリ(origin)にpushする。

## 備考

この手順は「コミットを作ってから、そのあとブランチを切る」という順序になっています。そのため、コミットした時点のブランチ（通常は `main`）にも、そのコミットがそのまま含まれることになります。先にブランチを分岐してから作業・コミットする一般的なPRフロー（ブランチを切ってから変更・コミットする方式）とは前提が異なるので、その点は認識しておいてください。
